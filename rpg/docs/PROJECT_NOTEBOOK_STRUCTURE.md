# Project Notebook Structure

This archived document outlines the intended structure for the inactive Google Drive / Gemini workflow experiment.

**Important Note:** The Google Drive documents are working memory only. They are *not* the source of truth for the project. The repository (`GitHub main`) and `rpg/docs/PROJECT_STATE_CURRENT.md` remain the definitive source of truth.

## Folder: `rpg-bot`

### Intended Documents:

1. **`01_Design_and_Balance_Ideas`**
   - **Purpose:** Scratchpad for the historical Design & Balance Gem. Used for brainstorming, theoretical scaling, class fantasy ideas, and balance philosophies before they are finalized.

2. **`02_Active_Decision_Packets`**
   - **Purpose:** Holds formulated Decision Packets that are ready for the historical Producer Gem to convert into implementation plans.

3. **`03_Implementation_Plans_Drafts`**
   - **Purpose:** Scratchpad for the historical Producer Gem. Used to draft PR scopes, Codex prompts, and testing requirements before they are finalized and handed off to Codex.

4. **`04_Review_Notes_and_Tails`**
   - **Purpose:** Used during the review phase to track cheap tails, blockers, and notes on Codex PRs.

5. **`05_Backlog_and_Future`**
   - **Purpose:** Unconfirmed, planned, or deferred ideas that might be picked up later.

*Note: Do not create these Google Drive files directly from the repository. This document exists so the historical Gemini workflow knew the expected structure when populating the Drive itself.*
