# Project Notebook Structure

This document outlines the intended structure for the Google Drive folder (`rpg-bot`) that serves as the working memory for Gemini.

**Important Note:** The Google Drive documents are working memory only. They are *not* the source of truth for the project. The repository (`GitHub main`) and `rpg/docs/PROJECT_STATE_CURRENT.md` remain the definitive source of truth.

## Folder: `rpg-bot`

### Intended Documents:

1. **`01_Design_and_Balance_Ideas`**
   - **Purpose:** Scratchpad for the Design & Balance Gem. Used for brainstorming, theoretical scaling, class fantasy ideas, and balance philosophies before they are finalized.

2. **`02_Active_Decision_Packets`**
   - **Purpose:** Holds formulated Decision Packets that are ready for the Producer Gem to convert into implementation plans.

3. **`03_Implementation_Plans_Drafts`**
   - **Purpose:** Scratchpad for the Producer Gem. Used to draft PR scopes, Jules prompts, and testing requirements before they are finalized and handed off to Jules.

4. **`04_Review_Notes_and_Tails`**
   - **Purpose:** Used during the review phase to track cheap tails, blockers, and notes on Jules' PRs.

5. **`05_Backlog_and_Future`**
   - **Purpose:** Unconfirmed, planned, or deferred ideas that might be picked up later.

*Note: Do not create these Google Drive files directly from the repository. This document exists so Gemini knows the expected structure when populating the Drive itself.*
