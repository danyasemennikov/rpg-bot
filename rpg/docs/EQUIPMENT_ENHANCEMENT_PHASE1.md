# EQUIPMENT_ENHANCEMENT_PHASE1.md

Документ фиксирует phase-1 правила заточки экипировки для instance-first модели (`gear_instances`).

## 1) Диапазон

- Заточка: `+0 .. +15`.
- Попытка всегда относится к конкретному gear instance.

## 2) Материалы по диапазонам

| Целевой уровень | Материал |
|---|---|
| `+1 -> +5` | `🧩 Осколок заточки` (`enhance_shard`) |
| `+6 -> +9` | `⚙️ Кристалл усиления` (`enhancement_crystal`) |
| `+10 -> +12` | `✴️ Эссенция мощи` (`power_essence`) |
| `+13 -> +15` | `🌑 Пепельный сердечник` (`ashen_core`) |

Базовые live-источники в текущем phase-1:
- `enhance_shard` — дроп с обычных мобов низкого тира.
- `enhancement_crystal` — дроп с мобов среднего тира.
- `power_essence` — дроп с сильных/элитных мобов.
- `ashen_core` — bounded shop fallback (высокий level gate + высокая цена) до полноценного raid/GvG контура.

## 2.1) Временный характер текущих источников

Текущие источники материалов в phase-1 — это **bootstrap/live-enablement** решение.
Они добавлены, чтобы механика заточки была реально играбельна уже сейчас, без ожидания
полного будущего overhaul по мобам, луту и контентным диапазонам.

В следующем проходе по расширению мобов / loot progression / drop routing
эти источники должны быть пересмотрены.

Желаемое долгосрочное направление:
- `enhance_shard` → обычные ранние мобы / low-tier фарм;
- `enhancement_crystal` → mid-tier мобы;
- `power_essence` → сильные элитки / high-tier PvE;
- `ashen_core` → elite/boss/event/raid/GvG-grade источники.

Текущая доступность `ashen_core` в деревенском магазине — это временный bounded fallback.
После появления корректных high-tier drop-источников этот fallback должен быть удалён
или существенно урезан.

Также будущий loot-pass должен увести материалы заточки от нескольких вручную назначенных мобов
к более системной модели распределения по tier/content-band.

## 3) Исходы попытки (phase-1)

| Цель | Успех | Неудача | Откат | Поломка |
|---|---:|---:|---:|---:|
| `+1 -> +3` | 80% | 20% | 0% | 0% |
| `+4 -> +6` | 55% | 30% | 15% | 0% |
| `+7 -> +9` | 30% | 35% | 30% | 5% |
| `+10` | 20% | 30% | 35% | 15% |
| `+11` | 15% | 25% | 35% | 25% |
| `+12` | 10% | 20% | 35% | 35% |
| `+13` | 7% | 18% | 30% | 45% |
| `+14` | 4% | 16% | 25% | 55% |
| `+15` | 2% | 13% | 20% | 65% |

Семантика исходов:
- **Успех**: `enhance_level += 1`.
- **Неудача**: уровень без изменений.
- **Откат**: `enhance_level -= 1` (минимум `0`).
- **Поломка**: instance уничтожается.

## 4) Атомарность

Одна попытка заточки выполняется атомарно:
1. проверка ресурсов (gold + материал),
2. списание ресурсов,
3. roll исхода,
4. применение outcome к instance-состоянию,
5. commit.

Никакой части попытки не должно существовать в полу-состоянии.
